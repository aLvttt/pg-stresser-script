# pg-stresser-script

Версия: `2.2`

## ВНИМАНИЕ: в данной версии из-за однопоточности генерации SQL-запросов при высоких нагрузках скрипт может не вписываться в заданное время работы или выдавать некорректное кол-во SQL-запросов!

`pg-stresser.sh` — это bash-скрипт для PostgreSQL. Он может:

- создать техническую тестовую зону;
- запустить в ней контролируемую SQL-нагрузку;
- удалить эту зону после проверки.

Скрипт строит весь рабочий процесс вокруг собственной автосозданной технической тестовой зоны.

---

## Как устроен wizard

Если запустить скрипт без аргументов:

```bash
./pg-stresser.sh
```

или явно:

```bash
./pg-stresser.sh --interactive
```

откроется wizard. В нём есть 3 раздела.

### 1. Генерация SQL-запросов

Это раздел для запуска нагрузки.

Когда вы выбираете этот пункт, скрипт:

- автоматически использует свою техническую тестовую зону;
- предлагает выбрать профиль нагрузки.

Внутри этого раздела доступны 3 варианта:

#### `test`

Это самый лёгкий и короткий профиль.

Он нужен, когда вы хотите быстро проверить:

- что скрипт вообще запускается;
- что PostgreSQL отвечает;
- что SQL-события доходят до системы наблюдения.

Параметры профиля:

- `120 SQL/min`
- `60 sec`

#### `base`

Это основной рабочий профиль.

Он нужен, когда вы хотите получить уже заметную и стабильную нагрузку, а не просто короткую проверку.

Параметры профиля:

- `5000 SQL/min`
- `300 sec`

#### `custom`

Это режим, в котором пользователь задаёт свои значения вручную.

Важно: в версии `2.2` здесь настраиваются только:

- количество SQL-запросов в минуту;
- продолжительность нагрузки в секундах.

Все остальные параметры автоматически берутся из профиля `base`.


### 2. Создание или пересоздание тестовой зоны

Это раздел подготовки окружения.

Его нужно использовать перед первым запуском нагрузки.

Когда вы выбираете этот пункт, скрипт:

- подключается как администратор PostgreSQL;
- создаёт технического пользователя;
- создаёт техническую базу;
- создаёт схему и таблицу;
- заполняет таблицу начальными тестовыми данными.

Этот режим также используется, если вы хотите начать заново с "чистой" тестовой зоны.

Важно понимать: это именно пересоздание, а не аккуратное обновление.

То есть скрипт может:

- удалить старую техническую базу;
- удалить старую техническую роль;
- создать всё заново.


### 3. Удаление тестовой зоны

Это раздел очистки.

Его используют после завершения проверки, когда техническая тестовая зона больше не нужна.

Когда вы выбираете этот пункт, скрипт:

- подключается как администратор PostgreSQL;
- удаляет техническую базу;
- удаляет технического пользователя.

Этот режим удобен, если вы хотите полностью убрать всё, что ранее создал скрипт.

## Ключи для быстрого запуска

Ниже перечислены ключи, которые нужны чаще всего.

### `--interactive`

Запускает wizard.

Пример:

```bash
./pg-stresser.sh --interactive
```

Используйте этот ключ, если хотите пройти всё пошагово и не запоминать параметры.

### `--mode prepare`

Создаёт или пересоздаёт тестовую зону.

Пример:

```bash
./pg-stresser.sh --mode prepare --admin-password secret
```

Когда использовать:

- при первом запуске;
- если хотите заново создать тестовую среду;
- если хотите начать работу с чистой тестовой зоной.

Что делает:

- создаёт техническую БД;
- создаёт технического пользователя;
- создаёт схему и таблицу;
- добавляет стартовые данные.

### `--mode run`

Запускает генерацию SQL-нагрузки.

Пример:

```bash
./pg-stresser.sh --mode run --preset test
```

Когда использовать:

- после `prepare`;
- когда тестовая зона уже существует;
- когда нужно выполнить сам прогон нагрузки.

Важно: сам по себе `--mode run` не выбирает параметры нагрузки. Обычно вместе с ним используется `--preset`, либо `--rpm` и `--duration`.

### `--mode delete`

Удаляет тестовую зону.

Пример:

```bash
./pg-stresser.sh --mode delete --admin-password secret
```

Когда использовать:

- после завершения теста;
- если больше не нужна техническая БД и технический пользователь;
- если хотите полностью очистить окружение.

### `--preset test`

Запускает короткий и лёгкий профиль нагрузки.

Пример:

```bash
./pg-stresser.sh --mode run --preset test
```

Что удобно знать:

- это самый безопасный старт;
- он подходит для первой проверки;
- с него проще всего начинать знакомство со скриптом.

### `--preset base`

Запускает основной базовый профиль нагрузки.

Пример:

```bash
./pg-stresser.sh --mode run --preset base
```

Что удобно знать:

- этот профиль заметно тяжелее `test`;
- он идёт дольше;
- его стоит использовать после того, как вы уже убедились, что тестовая зона работает нормально.

### `--rpm`

Задаёт количество SQL-запросов в минуту.

Пример:

```bash
./pg-stresser.sh --mode run --rpm 2000 --duration 180
```

Когда использовать:

- если стандартные профили не подходят;
- если нужно вручную задать темп нагрузки.

В текущей версии этот ключ особенно полезен вместе с `custom`-логикой wizard.

### `--duration`

Задаёт длительность прогона в секундах.

Пример:

```bash
./pg-stresser.sh --mode run --rpm 2000 --duration 180
```

Когда использовать:

- если нужно, чтобы прогон длился дольше или короче стандартного;
- если вы вручную задаёте параметры нагрузки.

### `--admin-password`

Передаёт пароль администратора PostgreSQL.

Пример:

```bash
./pg-stresser.sh --mode prepare --admin-password secret
```

Когда нужен:

- для `prepare`;
- для `delete`.

Почему он нужен:

- именно через административное подключение скрипт создаёт и удаляет техническую зону.

### `--host`

Задаёт путь к Unix socket PostgreSQL.

Пример:

```bash
./pg-stresser.sh --mode run --host /var/run/postgresql --preset test
```

По умолчанию используется:

```bash
/var/run/postgresql
```

Важно: здесь должен быть именно путь к socket-директории, а не `localhost` и не `127.0.0.1`.

### `--port`

Задаёт порт PostgreSQL.

Пример:

```bash
./pg-stresser.sh --mode run --port 5432 --preset test
```

Обычно менять его не нужно, если у вас стандартная установка PostgreSQL.

### `--database`

Задаёт имя технической тестовой базы.

Пример:

```bash
./pg-stresser.sh --mode prepare --database stresser_test --admin-password secret
```

По умолчанию:

```bash
stresser_test
```

Обычно этот ключ нужен только если вы хотите переименовать техническую зону.

### `--user`

Задаёт имя технического тестового пользователя.

Пример:

```bash
./pg-stresser.sh --mode prepare --user stresser_user --admin-password secret
```

По умолчанию:

```bash
stresser_user
```

### `--password`

Задаёт пароль технического тестового пользователя.

Пример:

```bash
./pg-stresser.sh --mode prepare --password stresser_pass --admin-password secret
```

Этот пользователь затем используется для запуска самой нагрузки.

### `--schema`

Задаёт схему тестовой зоны.

Пример:

```bash
./pg-stresser.sh --mode prepare --schema stresser_probe --admin-password secret
```

По умолчанию:

```bash
stresser_probe
```

### `--table`

Задаёт имя тестовой таблицы.

Пример:

```bash
./pg-stresser.sh --mode prepare --table sql_events --admin-password secret
```

По умолчанию:

```bash
sql_events
```

---

## Готовые команды

### 1. Первый запуск

```bash
./pg-stresser.sh --mode prepare --admin-password secret
./pg-stresser.sh --mode run --preset test
```

### 2. Базовый рабочий прогон

```bash
./pg-stresser.sh --mode run --preset base
```

### 3. Свой темп и своё время

```bash
./pg-stresser.sh --mode run --rpm 2500 --duration 120
```

### 4. Полная очистка после теста

```bash
./pg-stresser.sh --mode delete --admin-password secret
```

---

## Значения по умолчанию

Если вы ничего не меняете вручную, скрипт использует такие значения:

| Параметр | Значение |
|---|---|
| `PGHOST` | `/var/run/postgresql` |
| `PGPORT` | `5432` |
| `PGDATABASE` | `stresser_test` |
| `PGUSER` | `stresser_user` |
| `PGPASSWORD` | `stresser_pass` |
| `TEST_SCHEMA` | `stresser_probe` |
| `TEST_TABLE` | `sql_events` |

Административное подключение по умолчанию:

| Параметр | Значение |
|---|---|
| `PGADMIN_HOST` | такой же, как `PGHOST` |
| `PGADMIN_PORT` | такой же, как `PGPORT` |
| `PGADMIN_DATABASE` | `postgres` |
| `PGADMIN_USER` | `postgres` |
| `PGADMIN_PASSWORD` | пусто |

---

## Переменные окружения

Если не хотите передавать параметры в командной строке каждый раз, можно использовать переменные окружения:

```bash
export PGHOST=/var/run/postgresql
export PGPORT=5432
export PGDATABASE=stresser_test
export PGUSER=stresser_user
export PGPASSWORD=stresser_pass

export TEST_SCHEMA=stresser_probe
export TEST_TABLE=sql_events

export PGADMIN_HOST=/var/run/postgresql
export PGADMIN_PORT=5432
export PGADMIN_DATABASE=postgres
export PGADMIN_USER=postgres
export PGADMIN_PASSWORD=secret
```

После этого команды можно писать короче:

```bash
./pg-stresser.sh --mode prepare
./pg-stresser.sh --mode run --preset test
./pg-stresser.sh --mode delete
```

---

## Что важно помнить

- Сначала обычно идёт `prepare`, потом `run`, потом `delete`.
- `prepare` может удалить старую техническую тестовую зону и создать её заново.
- `run` запускает нагрузку в собственной технической тестовой зоне.
- Скрипт использует один постоянный `psql`-сеанс для предсказуемой генерации SQL-событий.
- Для подключения используется локальный Unix socket.

---

## Минимальная инструкция в двух строках

Если нужна совсем короткая версия:

```bash
./pg-stresser.sh --mode prepare --admin-password secret
./pg-stresser.sh --mode run --preset test
```

После завершения:

```bash
./pg-stresser.sh --mode delete --admin-password secret
```
